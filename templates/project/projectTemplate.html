Requirements

We'd like a single-page application written in BackboneJS, supported by the Twitter Bootstrap framework. In terms of data, it will make use of the riskmethods API (see https://developer.riskmethods.net/) and the API token 61dbf36d800b2f054361b75169ae1e10

OK 1) The app has two routes: List of vehicles and detail view of one vehicle. (No need for a login page, the token can be hardcoded.)

OK 2) At launch, the form will fetch the (first page of) vehicles from the API, or show an alert if the token was invalid.

OK 3) All vehicles are listed (paginated) with name and DUNS number and a large colored icon with a number inside with the Total Risk Score (score_card/risk_score in the JSON response). Low numbers should be green, high should be red indicating high risk.

OK 4) Clicking on a vehicle shows the detail view with the address and the names and scores of the individual Risk Groups (so one level more detail in the JSON) again colored.

OK 5) All vehicles should also have a star icon to select favorites, which should be shown both in the list and detail view.

OK 6) The list of favorites should be persisted using in-browser LocalStorage. (If you choose to use a Backbone extension for this, you're welcome to do so.)

OK 7) The app is supposed to run on desktop and mobile, so make sure to use responsive design to support both scenarios.

Should you have any question, feel free to let me know any time. And once done, please upload the app on GitHub and send us the link.
